## 1. 项目概述

### 1.1 目标

开发一款面向国内用户的跨平台 Git GUI 桌面工具，参考 SourceGit 的全部功能，并针对国内用户进行深度适配。

### 1.2 核心定位

- **全功能**: 覆盖 SourceGit 所有主要功能（80+ Git 操作）
- **国内适配**: Gitee 深度集成、国内 AI 服务、Git 镜像加速、多平台支持
- **轻量高效**: 基于 Tauri，体积约 8MB，内存占用低
- **跨平台**: 支持 Windows / macOS / Linux

### 1.3 技术栈

| 层次 | 技术 | 版本 |
|---|---|---|
| 桌面框架 | Tauri | 2.x |
| 前端框架 | React + TypeScript | React 18 |
| 状态管理 | Zustand | latest |
| UI 组件 | shadcn/ui + Tailwind CSS | latest |
| 提交图渲染 | HTML5 Canvas | - |
| 路由 | React Router | latest |
| 图标 | Lucide + 自定义 SVG | - |
| Git 进程管理 | `std::process::Command` (Rust) | - |
| 序列化 | serde + serde_json | - |
| 文件监听 | notify (Rust crate) | latest |
| HTTP 客户端 | reqwest (Rust crate) | latest |
| 错误处理 | thiserror (Rust crate) | latest |

## 3. Git CLI 桥接层

### 3.1 进程管理基类

所有 Git 操作通过 Rust `std::process::Command` 调用本地 git CLI。

**三种执行模式:**
- `exec_async()`: 异步执行，实时回调输出（clone/fetch/push/pull 等长时间操作）
- `read_to_end()`: 同步执行，读取全部 stdout/stderr（查询类操作）
- `read_to_end_async()`: 异步版本

**全局参数前缀:**
- `--no-pager`（禁用分页）
- `-c core.quotepath=off`（正确显示中文路径）
- `-c credential.helper=<平台适配>`

**编辑器劫持模式:**
- `None`: `core.editor=true`（空操作）
- `CoreEditor`: 注册为 `core.editor`（commit message 编辑）
- `RebaseEditor`: 注册为 `sequence.editor` + `core.editor`（交互式 rebase）

**SSH 支持:**
- 自动设置 `SSH_ASKPASS` 指向自身程序
- 支持每个远程独立 SSH 密钥配置
- `GIT_SSH_COMMAND` 注入指定密钥

**错误处理:**
```rust
pub enum GitError {
    ProcessError(String),
    CommandError { code: i32, message: String },
    ParseError(String),
    Timeout(String),
    Cancelled,
}
```

### 3.2 输出解析策略

| Git 命令 | 解析方式 | 格式 |
|---|---|---|
| `git log` | 自定义 format + `\0` 分隔 | `%H%x00%P%x00%D%x00%an%x00%ae%x00%at%x00%s` |
| `git status` | `--porcelain=v2` | 机器可读格式 |
| `git diff` | 逐行解析 unified diff | 标准 unified diff |
| `git blame` | `--porcelain` | 机器可读格式 |
| `git stash list` | `--format="..."` | 自定义格式 |
| `git branch/tag` | `--list --format="..."` | 自定义格式 |
| `git remote -v` | 逐行解析 | `name\turl\t(fetch/push)` |

## 4. 前端组件架构

### 4.1 状态管理 (Zustand)

| Store | 职责 |
|---|---|
| `repository-store` | 仓库列表、活跃仓库、标签页管理 |
| `git-store` | 每个仓库的 Git 数据（提交/分支/变更/stash 等） |
| `preferences-store` | 全局设置（主题/语言/AI/镜像/Gitee） |
| `ui-store` | UI 状态（命令面板/对话框/通知） |

### 4.2 核心页面

**RepositoryPage（仓库主页面）**: 三栏布局
- 左栏（200-500px 可拖拽）: 仪表盘/搜索切换 + 分支/标签/子模块/工作树树
- 主内容区: Histories / WorkingCopy / StashesPage 切换
- 进行中操作提示栏（merge/rebase/cherry-pick/revert 冲突）

**HistoriesPage（提交历史）**: 上下/左右可切换布局
- 上半: 提交列表 + Canvas 提交图叠加
- 下半: CommitDetail / RevisionCompare（DataTemplate 路由）

**WorkingCopyPage（工作区）**: 左右分栏
- 左: 未暂存/已暂存双区域（拖拽分隔）
- 右: DiffView + CommitMessageEditor + 提交操作栏

**WelcomePage（欢迎页）**: 三卡片布局
- 打开仓库 / 克隆仓库 / 初始化
- 最近仓库列表

### 4.3 提交图 (CommitGraph Canvas)

- 10 色圆盘循环分配路径颜色
- 曲线规则: 向右=二次贝塞尔, 向左=三次贝塞尔(4px偏移), 垂直=直线
- 节点类型: Default(圆点) / Head(双圆环) / Merge(十字)
- 高亮模式: All / CurrentBranchOnly / SelectedCommitsOnly / CurrentBranchAndSelectedCommits
- 虚拟化渲染: 仅绘制可见区域（PushClip + PushTransform）
- 与虚拟列表联动，Canvas 和列表行严格对齐

## 6. 核心 Git 操作

### 6.1 合并 (Merge)

- 策略: CreateMergeCommit (--no-ff) / FastForwardOnly (--ff-only) / FastForward (--ff) / Squash (--squash)
- 合并策略选项: recursive / octopus / ours / resolve
- 冲突处理: 检测冲突文件列表 → 内置合并编辑器或外部工具 → 解决后 `git add` + `git commit --no-edit`
- 进行中状态持久化

### 6.2 变基 (Rebase)

- 非交互式: `git rebase <onto>` + 冲突暂停/继续/跳过/终止
- 交互式: Todo 列表编辑器，支持 6 种操作 (pick/reword/edit/squash/fixup/drop)
- 可拖拽排序步骤
- `--preserve-merges` / `--autosquash` / `--ignore-whitespace` 选项
- 进度回调: 每步完成时通知前端

### 6.3 Cherry-pick

- 支持单个或多个提交
- `--no-commit` / `--signoff` / `--edit` 选项
- 冲突暂停: continue / skip / abort

### 6.4 提交 (Commit)

- `--signoff` / `--no-verify` / `--amend` / `--reset-author` / `--allow-empty`
- GPG 签名: `-S[<keyid>]`
- 修正模式: 加载上次提交消息作为默认值
- 消息写入临时文件通过 `-F` 传递

### 6.5 推送 (Push)

- `--force` / `--force-with-lease` / `--set-upstream` / `--tags` / `--follow-tags`
- 镜像加速: 自动替换 remote URL
- 实时进度回调 (Tauri Event)

### 6.6 拉取 (Pull)

- `--rebase` / `--ff-only` / `--autostash`
- 自动检测当前分支上游

### 6.7 获取 (Fetch)

- `--prune` / `--tags` / `--no-tags` / `--depth <n>` (浅获取)

### 6.8 其他操作

| 操作 | Git 命令 |
|---|---|
| Reset | `git reset --soft/--mixed/--hard <sha>` |
| Revert | `git revert <sha>` |
| Stash push/pop/apply/drop/clear/show | `git stash` 系列 |
| Tag create/delete/push | `git tag` 系列 |
| Remote add/edit/delete/prune | `git remote` 系列 |
| Submodule add/update/deinit/set-branch | `git submodule` 系列 |
| Worktree add/remove/prune/list | `git worktree` 系列 |
| Blame | `git blame --porcelain` |
| LFS track/untrack/fetch/pull/push/prune/lock/unlock | `git lfs` 系列 |
| Archive | `git archive` |
| GC | `git gc` |
| Bisect start/good/bad/skip/reset/log | `git bisect` 系列 |
| GitFlow init/start/finish | `git flow` 系列 |
| Patch save/apply | `git format-patch` / `git apply` |
| Assume Unchanged | `git update-index --assume-unchanged` |
| Skip Worktree | `git update-index --skip-worktree` |

## 7. Diff 视图

### 7.1 文本 Diff

- 语法高亮 (TextMate Grammars)
- 并排对比 (SideBySide)
- 自动换行 (WordWrap)
- 忽略空白变更
- 隐藏字符显示
- 变更块导航: 首个/上一个/下一个/最后一个 (Ctrl+Alt+Home/Up/Down/End)
- 上下文行数调节
- 外部合并工具 (Ctrl+Shift+D)
- 行变更统计 (+N / -N)

### 7.2 图片 Diff (4 种模式)

- SideBySide: 左右并排，显示尺寸和文件大小
- Swipe: 滑动分割线对比
- Blend: Alpha 混合滑块 (0~1)
- Difference: 差异程度滑块

### 7.3 其他 Diff 类型

- BinaryDiff: 显示新旧文件大小
- LFSDiff: LFS 指针信息
- SubmoduleDiff: 新旧提交 + 未提交变更数 + 打开子模块对比

### 27.8 动效规范（"平静界面"原则）

- **克制**: 动效仅用于解释结构（刚刚发生了什么、正在发生什么、接下来会发生什么）
- **时长**: 快速操作 100ms / 过渡 200ms / 展开收起 300ms
- **缓动**: `ease-out`（进入） / `ease-in`（退出） / `ease-in-out`（状态切换）
- **避免**: 装饰性动画、弹跳效果、过度粒子效果

### 27.10 深色模式最佳实践（遵循 Material Design 标准）

- **永远不用纯黑** `#000000` — 使用 `#1e1f23` 避免光晕效应
- **正文对比度 15.8:1** — 通过 87% 不透明度白色文字实现
- **次要文字不低于 38% 不透明度** — 确保可读性
